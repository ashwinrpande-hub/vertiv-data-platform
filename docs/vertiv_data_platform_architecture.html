⚡ Performance Considerations
Clustering · Search Optimization · Resource Governance · Latency Design · Scaling Strategy
Bronze Clustering
▸
SALES_ORDERS: CLUSTER BY (LOAD_DATE, ORDER_DATE) — time-travel queries and incremental loads skip most micro-partitions
▸
ORDER_ITEMS: CLUSTER BY (LOAD_DATE, ORDER_ID) — ORDER_ID joins prune heavily
▸
CUSTOMERS: CLUSTER BY (LOAD_DATE) — batch loads are date-sequential
▸
Kafka events: CLUSTER BY DATE(EVENT_TIMESTAMP) — real-time queries filter by day
Silver Clustering
▸
CUSTOMER: CLUSTER BY (CUSTOMER_HK) — point lookups by hash key are O(1)
▸
PRODUCT: CLUSTER BY (PRODUCT_HK)
▸
SALES_ORDER: CLUSTER BY (CUSTOMER_HK, DATE(ORDER_DATE)) — customer-scoped + time range queries
▸
ORDER_LINE: CLUSTER BY (ORDER_HK) — header-to-line joins
Gold Clustering
▸
FACT_SALES_ORDERS: implicit via Dynamic Table (inherits Silver clustering)
▸
ML Features: CLUSTER BY (AS_OF_DATE, CUSTOMER_HK) — PIT feature retrieval for model training
▸
MODEL_PREDICTIONS: CLUSTER BY (MODEL_NAME, AS_OF_DATE)
Search Optimization
▸
DIM_CUSTOMER: EQUALITY(CUSTOMER_NAME, COUNTRY_CODE, MASTER_CUSTOMER_HK) — point lookups from BI tools
▸
DIM_PRODUCT: EQUALITY(PRODUCT_CODE, PRODUCT_NAME, PRODUCT_FAMILY)
▸
Avoids full table scan for "WHERE PRODUCT_CODE = 'UPS-GXT5-10K'"
Warehouse Sizing
▸
INGEST_WH XSmall — Snowpipe is serverless; only DQ tasks need compute
▸
TRANSFORM_WH Small — Dynamic Tables: incremental, low memory
▸
ANALYTICS_WH Medium multi-cluster 1-3 — concurrency scaling for BI, ECONOMY policy saves credits
▸
ML_WH Large — 60s auto-suspend; Snowpark needs RAM for pandas/scikit-learn
Cost Governance
▸
Resource Monitor VERTIV_DEMO_GUARD: 300 credits/month
▸
Notify at 75% → 90% → Suspend at 100%
▸
Applied to ANALYTICS_WH + ML_WH (expensive warehouses only)
▸
Bronze: 7d retention. Silver: 30d. Gold/Products: 90d — cost-tiered
Latency Design
▸
Kafka → Bronze: near real-time via Snowpipe Streaming
▸
Bronze → Silver Dynamic Tables: 5-minute lag
▸
Silver → Gold Fact: 15-minute lag
▸
SLO freshness target: ≤60 minutes end-to-end
▸
SLO P95 query: ≤5 seconds
Materialized Views
▸
MV_REGIONAL_SALES_DAILY: pre-aggregated by SOURCE_SYSTEM, REGION, DATE — eliminates GROUP BY on FACT for most BI queries
▸
MV_PRODUCT_REVENUE_MONTHLY: PRODUCT_SK × SOURCE × DATE — avoids cross-dim joins for product revenue reports
▸
Snowflake auto-maintains MVs when base tables change
Data Quality Performance
▸
DQ Framework Task runs every 6h on TRANSFORM_WH — off-peak
▸
HASH_DIFF change detection avoids full row comparisons — single SHA256 comparison per row
▸
SLI measurement task: every 30min on INGEST_WH (XSmall) — lightweight aggregation only
▸
Quarantine table: rejected records stored as VARIANT — forensic replay without schema dependency
RBAC & Row Policies
▸
RBAC schema-level boundaries: SAP_EUROPE_READER can only see SAP_EUROPE schema (PII protection)
▸
Row Access Policy on Bronze: regional readers filtered to their own region's rows
▸
No performance impact on Global Analyst role (no row filter applied)
▸
Data Products DB (VERTIV_PRODUCTS): Secure Views strip PII before Marketplace listing
